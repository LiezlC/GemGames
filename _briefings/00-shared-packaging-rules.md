# Shared packaging rules (applies to all four umbrella briefings)

These rules are embedded by reference in each of the four umbrella briefings. Every requirement below is **mandatory** and must be reflected in the consolidated app's delivered artifact. The executing agent does not need to receive this file separately — the relevant rules are restated inline in each briefing.

This file exists as the single source of truth so that if a rule changes, it updates across all four umbrellas from one edit.

---

## 1. AI-generation disclosure (non-negotiable)

Every consolidated umbrella app must include, visibly in its footer and in a meta description tag, the following disclosure:

> Interactive applets originally generated by Google Gemini from source images. Curated, consolidated, and extended by Liezl Coetzee.

**Why:** transparency is a buyer-trust prerequisite on Gumroad and itch.io, and it's correct. The consolidation work (the shell, the layering, the cross-applet mechanics, the visual extension) is the human work. The generation of the individual applets was AI. Both deserve attribution.

**Where:** footer of the shell page (always visible), `<meta name="description">`, and a small "About" panel accessible from the shell chrome.

---

## 2. Source-image usage (the core design constraint)

Each source applet in `.../art-mindfulness-gumroad-bundle/.../` and `.../speculative-ai-futures/.../` and `ArtApps/**/` has a `source.jpg` or `source.png` (or occasionally `.pdf`) alongside its `index.html`. That image is **what was given to Gemini to inspire the applet**.

In the consolidated umbrella app, the source image must be **actively used**, not just stored. Minimum viable uses:

- **Visible as part of the applet selection UI.** Each applet tile/card in the shell shows its source image. This is baseline.
- **Used as visual context when an applet is active.** The source image should be visible, layered, or referenced in the applet viewport — as a parallax background, a "source of inspiration" sidebar, a ghost-image watermark, or something equivalent. The point is that the user sees the provenance.
- **Extended where the applet's mechanic supports it.** If the applet manipulates pixels, the source image should be one of the available inputs. If the applet generates a scene, the source image's palette should influence that scene. Per-umbrella creativity expected here.

If a source file is a PDF instead of an image (rare — check for it), extract the first page as an image at build time and use that as the source image. Keep the original PDF accessible from the applet's "about" panel.

---

## 3. Author metadata in every HTML file

Every HTML file the umbrella ships (shell + each embedded applet) must include:

```html
<meta name="author" content="Liezl Coetzee">
<meta name="description" content="[applet or shell description] — part of the [UMBRELLA NAME] collection. Interactive applets originally generated by Google Gemini from source images; curated, consolidated, and extended by Liezl Coetzee.">
<meta name="generator" content="Gemini (original applets); human consolidation by Liezl Coetzee">
```

---

## 4. Technical constraints

- **Single HTML file** for the shell. All shell CSS/JS inlined or loaded from CDN with SRI hashes. No build step, no bundler, no npm install.
- **Each applet preserved as its own HTML** in a subfolder. Shell loads applets in `<iframe>` or via `fetch`-inject, depending on the unifying-mechanic design per umbrella. Do NOT rewrite or refactor the individual applets' internals — respect the original Gemini output.
- **CDN allowed** for common libraries (Tailwind Play CDN, p5.js, three.js, GSAP, etc.) **only if** the shell genuinely needs them. Avoid adding heavyweight libraries if vanilla Canvas/DOM works.
- **No external backend.** Everything must work offline after first load.
- **No external analytics / trackers / fingerprinting.** Gumroad/itch.io buyers expect self-contained, privacy-respecting artifacts.
- **Mobile-responsive down to 375px width.** Touch controls where a mouse interaction exists.
- **Accessibility floor:** keyboard navigation of the applet gallery, visible focus rings, sufficient contrast on all text, alt text on every image including source images.

---

## 5. Curation authority

Each briefing specifies a source pool of N applets. The executing agent is **explicitly authorised to skip or exclude** applets that:

- Fail to load / contain broken code.
- Are thematically off-concept for the umbrella.
- Duplicate another applet in the same umbrella (keep the better one, flag the skip in the README).
- Rely on copyrighted imagery the source image inadvertently contains.

Skipped applets must be listed in the output `README.md` under a "Skipped applets and why" section, so the decision is auditable.

---

## 6. Output structure

Every umbrella build delivers this structure into its target directory:

```
_builds/<umbrella-slug>/
├── index.html                  # the shell
├── README.md                   # author-facing notes
├── assets/
│   ├── fonts/ (if any)
│   ├── shell.css (if externalised)
│   └── shell.js  (if externalised)
└── applets/
    ├── <applet-slug>/
    │   ├── index.html          # the original applet HTML, unchanged
    │   └── source.{jpg,png,pdf}
    └── ...
```

The shell's applet-discovery logic reads from `applets/` — so adding an applet later is drag-and-drop, no shell code changes.

---

## 7. Acceptance criteria (the agent self-verifies before declaring done)

- [ ] Shell `index.html` opens in a browser (agent uses Playwright, Puppeteer, or headless Chrome to confirm) and renders without console errors.
- [ ] All source-pool applets that were not explicitly skipped appear in the shell's applet gallery.
- [ ] Each applet loads when selected from the gallery.
- [ ] Each applet tile displays its source image.
- [ ] Footer disclosure is visible on the shell and on every applet wrapper.
- [ ] Meta tags present on every HTML file.
- [ ] Mobile viewport (375×812) renders usable UI — agent verifies in headless mode at that resolution.
- [ ] No requests to external trackers or analytics in the network log.
- [ ] README.md in the output root lists: included applets (alphabetical), skipped applets (with reason), shell mechanic description, known issues.
- [ ] Total build folder size under 50 MB (source images dominate — flag if over, don't fail).

---

## 8. What NOT to do

- Do not rewrite the original Gemini-generated applet code to "improve" it. Preserve it as-is except for the minimum HTML-head metadata additions required by rule 3.
- Do not add copyrighted imagery, third-party illustrations, or stock photography beyond what's already in the source folders.
- Do not introduce tracking, analytics, or "phone-home" behaviour.
- Do not invent a pricing page, checkout flow, or marketing copy — that's out of scope. The shell should have a single "About" panel, no commerce UI.
- Do not commit the output to git — these are builds, not source. The source lives in the original applet folders. Add a `.gitignore` entry for `_builds/` if one doesn't already exist.
