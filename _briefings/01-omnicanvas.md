### C. Source-image usage (audit + extend)

The existing shell may or may not actively use source images. Audit, then ensure (at minimum):

- **Layer palette / picker** — every layer in the layers panel displays its source image as a circular thumbnail.
- **Inspiration panel** — a collapsible panel reachable from the shell chrome shows the source images for all currently-active layers, each with a caption naming the applet.
- **Seed-from-source effect** — add a shell-level button "Seed palette from source" that reads the colour histogram of the currently-topmost layer's source image and applies it as a palette tint across all layers (via CSS filter or Canvas composition). Subtle effect; user-reversible.

If any of these already exist, leave them. If they don't, add them — they're the core "source images as active material" requirement.

### D. About panel

The existing shell may not have one. Add an About panel, reachable from a small palette-icon in the top-left. Displays:

- OmniCanvas name + one-sentence description.
- The full AI-generation disclosure.
- A gallery of every included applet's source image, each with its applet name.
- Credit line: "Curated, consolidated, and extended by Liezl Coetzee · sociable.systems".
