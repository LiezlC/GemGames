## 🔧 How the Extraction Works

The `extract_artifacts.py` script:

1. Reads each JSON file from `jsons/` directory
2. Parses the JSON structure (id, name, html, originalImage, timestamp)
3. Extracts and decodes base64-encoded images/PDFs
4. Saves HTML files as `index.html` in project folders
5. Saves images/PDFs with proper file extensions
6. Generates a comprehensive gallery index page

### Supported Formats

- **Images**: PNG, JPG, WEBP, GIF, SVG
- **Documents**: PDF
- **Data URIs**: `data:image/*;base64,...` and `data:application/pdf;base64,...`

## 🛠️ Technical Details

### JSON Structure
```json
{
  "id": "UUID",
  "name": "original-filename.png",
  "html": "<!DOCTYPE html>...",
  "originalImage": "data:image/png;base64,...",
  "timestamp": "2025-11-21T01:15:30.864Z"
}
```

### File Validation

The extraction script validates files using magic number signatures:
- **PNG**: `\x89PNG\r\n\x1a\n`
- **JPG**: `\xFF\xD8\xFF`
- **PDF**: `%PDF`
- **WEBP**: `RIFF...WEBP`
- **GIF**: `GIF87a` or `GIF89a`

## 📦 Dependencies

## 🐛 Troubleshooting

### Images won't open
All 107 images have been validated and have correct magic number signatures. If an image won't open:
1. Check your image viewer supports the format (PNG, JPG, WEBP)
2. Try opening in a different application
3. Verify the file size is > 0 bytes

### HTML games don't work
- Ensure you're opening the files in a modern web browser (Chrome, Firefox, Safari, Edge)
- Some games may require JavaScript to be enabled
- Check browser console for any error messages

### Re-run extraction
If you need to re-extract all files:
```bash
python3 extract_artifacts.py
```

The script will overwrite existing files with fresh extractions.
